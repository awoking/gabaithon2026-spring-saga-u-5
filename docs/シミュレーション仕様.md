# SAGA-U 微生物進化シミュレーション 詳細仕様書

## 1. シミュレーション概要

### 1.1 目的
微生物集団の進化動態をリアルタイムで可視化するシミュレーター。複数株の競合、突然変異、水平遺伝子伝達（HGT）を含む。

### 1.2 基本アーキテクチャ
- **数値積分**: 4次ルンゲ・クッタ法（RK4）による常微分方程式（ODE）の解法
- **最適化**: Numba JITコンパイルによる高速化
- **並列処理**: 全株を同時に計算（ベクトル化）
- **イベント駆動**: RK4バッチ間で離散イベント（分裂、HGT、突然変異）を処理

---

## 2. 数値積分システム

### 2.1 RK4（4次ルンゲ・クッタ法）

#### 基本式
```
y_{n+1} = y_n + (h/6)(k1 + 2k2 + 2k3 + k4)

k1 = f(t_n, y_n)
k2 = f(t_n + h/2, y_n + h*k1/2)
k3 = f(t_n + h/2, y_n + h*k2/2)
k4 = f(t_n + h, y_n + h*k3)
```

#### 適用変数
- **N**: 各株の個体数ベクトル（状態変数）
- **S**: 基質濃度（スカラー）
- **T**: 毒素濃度（スカラー）
- **pH**: 環境pH（スカラー）

#### パラメータ
| パラメータ | デフォルト値 | 説明 |
|-----------|-------------|------|
| `dt` | 0.01 | 時間刻み（可変、adaptive dt使用時） |
| `batch_size` | 10,000 | 1バッチあたりのステップ数 |
| `dt_min` | 0.001 | adaptive dtの最小値 |
| `dt_max` | 0.02 | adaptive dtの最大値 |

### 2.2 Adaptive dt（適応的時間刻み）

#### 計算式
```python
dt_by_n = max_rel_change_per_step / max(|net_rate|)
dt_by_s = max_abs_s_change_per_step / |dS/dt|
dt = max(dt_min, min(dt_max, dt_by_n, dt_by_s))
```

#### パラメータ
- `max_rel_change_per_step`: 0.05（Nの最大相対変化率 5%/step）
- `max_abs_s_change_per_step`: 0.05（Sの最大絶対変化量）
- `adaptive_dt_enabled`: True/False

## 3. 常微分方程式（ODE）系

### 3.1 個体数変化（dN/dt）

```
dN_i/dt = (g_i - m_i - stress_i - D) * N_i
```

#### 成長率 g_i（モノド式ベース）
```
g_i = μ_max,i * (S / (K_s,i + S)) * f_temp * f_pH

f_temp = exp(-((temp - T_opt,i)^2) / 2)
f_pH = exp(-((pH - pH_opt,i)^2) / 2)
```

| 変数 | 説明 |
|------|------|
| `μ_max,i` | 株iの最大成長速度 |
| `K_s,i` | 株iのモノド定数（半飽和定数） |
| `S` | 基質濃度 |
| `temp` | 環境温度 |
| `T_opt,i` | 株iの最適温度 |
| `pH` | 環境pH |
| `pH_opt,i` | 株iの最適pH |

#### 維持コスト m_i（7項目加重和）
```
m_i = m_base + w1*μ_max + w2/K_s + w3*p + w4*r + w5*|T_opt - 25| + w6*|pH_opt - 7| + w7*Rad_res

重み: w1=0.05, w2=0.01, w3=0.02, w4=0.02, w5=0.005, w6=0.005, w7=0.08
m_base = 0.05
```

#### ストレス項 stress_i
```
stress_i = d_tox + d_rad

d_tox = k_tox * max(0, T - r_i)  # 毒素ストレス
d_rad = k_rad * max(0, rad - Rad_res_i)  # 放射線ストレス
```

#### 希釈項 D
```
D: 連続供給による希釈率（chemostat型）
D > 0 のとき wash-out 効果が発生
```

### 3.2 基質変化（dS/dt）

```
dS/dt = -Σ(g_i * N_i) / Y + D * (S_in - S)
```

| 変数 | 説明 |
|------|------|
| `Y` | 収率（biomass/substrate） |
| `D` | 希釈率 |
| `S_in` | 流入基質濃度 |

**離散的自動供給（auto_feed）との併用**:
- RK4バッチ内: 連続項 `D*(S_in - S)` のみ
- バッチ間: `S = min(S + feed_per_batch, feed_max_s)`

### 3.3 毒素変化（dT/dt）

```
dT/dt = Σ(p_i * N_i) - d_T * T - D * T
```

| 変数 | 説明 |
|------|------|
| `p_i` | 株iの毒素生産能力 |
| `d_T` | 毒素自然減衰率 |
| `D` | 希釈による除去 |

### 3.4 pH変化（dpH/dt）

```
dpH/dt = -k_acid * Σ(g_i * N_i)
```

| 変数 | 説明 |
|------|------|
| `k_acid` | 酸性化係数（代謝による酸生成） |

```python
# 各分裂時に確率的に発生（娘株のみ）

new_traits = max(0.01, new_traits)  # 下限クリッピング
new_traits[4] = clip(new_traits[4], 10, 50)  # T_opt
new_traits[5] = clip(new_traits[5], 3, 11)   # pH_opt
```

## 6. 離散イベントシステム

### 6.1 分裂イベント（Division）

#### トリガー条件
```python
if N[i] >= threshold:  # デフォルト threshold=50.0
    division_trigger()
```

#### 処理内容
```python
# 親株を半分に分割
N[parent] = N[parent] / 2.0

# 娘株を新規生成
daughter_idx = spawn(parent_idx=parent, initial_N=N[parent])
traits[daughter] = traits[parent] + mutation()
```

#### 制約
- 分裂は1バッチに1回のみ（最初に閾値を超えた株）
- free_indices（空きスロット）が必要
- 娘株は突然変異を伴う

### 6.2 水平遺伝子伝達（HGT）

#### 2つの実装方式

##### 方式A: カーネル内HGT（プラスミドプール）
```python
# RK4カーネル内で各ステップ実行
if random() < hgt_prob:
    pool_idx = random_choice(pool_size)
    if pool_conc[pool_idx] > threshold:
        # プラスミド形質 [p, r, Rad_res] を部分取り込み
        traits[i][2] += plasmid_pool[pool_idx][0] * 0.01
        traits[i][3] += plasmid_pool[pool_idx][1] * 0.01
        traits[i][6] += plasmid_pool[pool_idx][2] * 0.01
```

##### 方式B: マネージャー側HGT（株間伝達）
```python
# バッチ間で実行
hgt_rate = k_hgt * dt * batch_size
for i, j in donor_recipient_pairs:
    if random() < hgt_rate * N[i] * N[j]:
        new_idx = spawn(initial=False)
        # ドナーとレシピエントの形質を平均
        traits[new] = (traits[i] + traits[j]) / 2 + mutation()
        N[new] = 1.0
```

#### プラスミドプール更新
```python
# 10,000ステップごと
pool_conc *= 0.7  # 減衰
top_strain = argmax(N)
plasmid_pool[slot] = traits[top_strain][2:5, 6]  # [p, r, Rad_res]
pool_conc[slot] = 1.0
```

### 6.3 絶滅処理（Reap）

```python
# 毎バッチ終了後
for i in active_strains:
    if N[i] < 1.0:
        active_mask[i] = False
        N[i] = 0.0
        free_indices.append(i)
```

## 9. 実装の注意点

### 9.1 Numba JITの制約
- numpy配列のみ使用可能（リスト、辞書は不可）
- 動的な配列サイズ変更は避ける
- `@njit` デコレータを使用（nopython mode）

### 9.2 精度とパフォーマンスのトレードオフ
| 項目 | 設定 | トレードオフ |
|------|------|-------------|
| `dt` | 小さい | 精度↑、速度↓ |
| `batch_size` | 大きい | 通信効率↑、リアルタイム性↓ |
| `max_strains` | 大きい | 多様性↑、メモリ↑ |

### 9.3 数値安定性

#### ゼロ除算回避
```python
g = mu_max * (S / (Ks + S + 1e-8))  # Ks=0でも安全
m = w2 / (Ks + 0.01)  # 維持コスト計算
```

#### 負値クリッピング
```python
S = max(0.0, S)  # 基質負値を防ぐ
N = max(0.0, N)  # 個体数負値を防ぐ
```

#### オーバーフロー検出
```python
if not np.isfinite(N_total):
    # 発散検出、シミュレーション停止
```

## 10. パラメータチューニングガイド

### 10.1 目的別推奨設定

#### 長期安全運用（デモ・教育）
- 現在のデフォルトプリセットを使用
- `Y` を高く（100以上）
- `auto_feed_enabled = True`
- `D = 0`（離散供給のみ）

#### 競合ダイナミクス研究
- `Y` を低め（10-50）
- 初期株を複数用意（異なる `mu_max`, `Ks`）
- `D > 0`で連続供給（chemostat条件）

#### 進化実験
- `mutation_rate` を調整（`σ=0.01` がデフォルト）
- `hgt_prob` を上げる（0.01-0.05）
- 環境変動を導入（`temp`, `rad` を動的変更）

#### ストレス応答
- `k_tox`, `k_rad` を上げる
- 初期株に `r`, `Rad_res` を持たせる
- 段階的にストレスを増加

### 10.2 トラブルシューティング

| 症状 | 原因候補 | 対策 |
|------|---------|------|
| 即座に絶滅 | `Y` が低い、`D` が高い | `Y` を100以上に、`D=0`に |
| N が発散 | `mu_max` が高すぎる | 0.5以下に制限 |
| S が枯渇 | 供給不足 | `feed_per_batch` を増やす |
| pH が極端 | `k_acid` が高い | 0.0に設定（または低減） |
| 多様性なし | 突然変異率低い | `σ` を0.02-0.05に |
| HGT発生しない | `hgt_prob` が低い | 0.01以上に |
